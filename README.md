# Reputation System API

Тестовое задание: API системы перевода репутации между пользователями на Django REST Framework.


так же я добавил для удобства - Docker, posterSQL, Swagger 
##  Как запустить проект

Для быстрого запуска и тестирования в репозиторий уже включен файл `.env` с демонстрационными настройками.

1) Клонируйте репозиторий
   ```
   git clone https://github.com/BugganeHuman/reputation-system-api
2) Запустите Docker-контейнеры  
    ```
    docker compose up --build 
    ```
3) Выполните миграции базы данных:  
    ```
    docker compose exec web python manage.py migrate  
    ```

4) Создайте учетную запись суперюзера:  
    ```
    docker compose exec web python manage.py createsuperuser
    ```

## Создание тестовых данных
1) Перейдите в админку Django: http://localhost:8000/admin/ и войдите под созданным суперюзером.

2) Создайте одного или нескольких обычных юзеров на эндпоинте Users /admin/auth/user/)

При создании пользователя автоматически генерируется связанный профиль со стартовым балансом репутации (100) с помощью Django Signals. Убедиться в этом можно в разделе Profiles /admin/users/profile/.

## Выполнение основных сценариев
1) В проекте есть интерактивная документация Swagger. Перейдите по адресу: http://localhost:8000/api/docs/

### Сценарий 1: Авторизация в API
1) В интерфейсе Swagger найдите эндпоинт авторизации /api/users/auth/login/

2) Передайте логин и пароль созданного пользователя, получите access токен.

3) Нажмите кнопку Authorize в правом верхнем углу Swagger и вставьте access токен.

### Сценарий 2: Получение списка профилей (Узнать ID получателя)
1) Выполните GET-запрос на эндпоинт получения профилей, чтобы увидеть список доступных участников системы.

2) Скопируйте id профиля пользователя, которому вы хотите перевести репутацию.

### Сценарий 3: Создание транзакции (Перевод репутации)
1) Отправьте POST-запрос на эндпоинт /api/transactions/transaction/.

2) В теле запроса (body) передайте JSON:

```
{
  "receiver": "ID_получателя",
  "amount": "25"
}
```
